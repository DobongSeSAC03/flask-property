# [웹 시스템 성과 및 평가 보고서] 부동산 가치 분석 플랫폼

**작성일**: 2025.12.19
**프로젝트명**: 서울시 부동산 분석 웹 서비스 (Flask & Bootstrap Based)

---

## 1. 프로젝트 개요 (Web System Overview)

본 프로젝트는 복잡한 부동산 데이터를 일반 사용자가 직관적으로 이해할 수 있도록 **시각화된 웹 대시보드**로 제공하는 것을 목표로 하였습니다. 사용자 경험(UX) 최적화와 시스템 안정성에 중점을 두어 누구나 쉽게 접근 가능한 **반응형 웹 플랫폼**을 구축하였습니다.

### ✅ 주요 달성 목표
1.  **사용자 친화적 UI/UX**: 복잡한 수치 데이터를 그래프와 요약 테이블로 시각화하여 정보 전달력을 극대화함.
2.  **동적 데이터 처리**: 검색 조건에 따라 실시간으로 데이터를 집계하고 화면을 갱신하는 반응형 서비스 구현.
3.  **시스템 확장성 확보**: 모듈화된 Flask 아키텍처와 Bootstrap 그리드 시스템을 도입하여 유지보수 용이성 확보.

---

## 2. 웹 서비스 구축 성과 상세 (Key Achievements)

### ① 메인 대시보드 (`index.html`)
*   **성과**: 종합 상황판(Control Tower) 기능 구현.
*   **상세 내용**:
    *   **Chart.js 연동**: '자치구별 평균 평당가' 등 핵심 지표를 막대/선 그래프로 시각화하여 지역별 시세 흐름을 한눈에 파악 가능하게 함.
    *   **Bootstrap 그리드 적용**: 다양한 해상도(PC, 태블릿)에서도 차트와 테이블이 깨지지 않고 최적화되어 표시되도록 `col-sm-6` 기반의 반응형 레이아웃을 완성함.

### ② 예측 및 조회 시스템 (`predict.html`)
*   **성과**: 데이터 탐색 경험(Exploratory Data Analysis)의 고도화.
*   **상세 내용**:
    *   **2-Column Layout**: 화면을 '좌측 검색 패널'과 '우측 결과 패널'로 논리적으로 분리하여, 사용자가 검색 조건을 변경하며 즉각적으로 결과를 비교할 수 있는 효율적인 동선을 설계함.
    *   **고급 필터링**: 단순 지역 검색을 넘어 **'자본금(Amount)'**과 **'건물 유형(아파트/빌라)'** 필터를 추가하여, 사용자의 예산과 선호도에 맞는 맞춤형 매물 탐색 기능을 제공함.
    *   **Client-Side Aggregation**: 서버 부하를 줄이고 응답 속도를 높이기 위해, 조회된 데이터의 **'법정동별 거래량'** 및 **'평당가 집계'**를 브라우저(JavaScript)단에서 즉시 처리하여 보여줌.

### ③ 공통 인터페이스 및 아키텍처 (`base.html` & Backend)
*   **성과**: 표준화된 레이아웃 및 RESTful 구조 정립.
*   **상세 내용**:
    *   **Fixed Navigation**: 헤더(`fixed-top`)와 푸터(`fixed-bottom`)를 고정하여 긴 스크롤이 발생하는 데이터 페이지에서도 메뉴 접근성을 유지함.
    *   **API 설계**: 서브밋(Submit) 방식이 아닌 **AJAX 비동기 통신**(`POST /predict/location`)을 채택하여, 페이지 새로고침 없는 부드러운 사용자 경험(SPA급 UX)을 구현함.
    *   **ORM 최적화**: SQLAlchemy를 활용하여 데이터베이스 쿼리를 객체 지향적으로 관리하고, SQL Injection 등 보안 위협을 원천 차단함.

## 3. 기술적 평가 (Technical Evaluation)

### [UI/UX] 직관성 및 가독성 (usability)
*   일반적인 표(Table) 나열 방식에서 탈피하여, **통계 요약(Summary Card)**과 **시각화(Chart)**를 상단에 배치하는 'Dashboard First' 전략이 유효했음.
*   검색 결과 테이블에 건물 유형별 뱃지(Badge) 스타일링을 적용하여 시인성을 높임.

### [Performance] 응답 속도 및 효율성
*   복잡한 연산(집계)을 필요한 시점에만 수행하도록 설계하여 초기 로딩 속도를 최적화함.
*   정적 리소스(CSS, JS)와 동적 데이터(API JSON)의 전송을 분리하여 네트워크 효율성을 높임.

## 4. 웹 시스템 고도화 제안 (Future Improvements)

현재 구축된 '조회 및 분석' 중심의 시스템을 사용자 참여형 '플랫폼'으로 확장하기 위한 로드맵입니다.

1.  **사용자 개인화 (Personalization)**:
    *   로그인/회원가입 기능을 도입하여 '관심 지역(Favorite)' 등록 및 '최근 검색 조건' 저장 기능 제공.
2.  **지도 기반 시각화 (GIS Integration)**:
    *   현재의 텍스트/차트 기반 정보를 넘어, **Kakao/Naver 지도 API**를 연동하여 지도 위에 매물 위치와 주차장 분포를 마커로 표시하는 기능 개발.
3.  **반응형 모바일 최적화 (Mobile-First)**:
    *   모바일 환경에서의 터치 인터페이스를 고려하여, 필터 패널을 '슬라이드 메뉴(Offcanvas)' 형태로 변환하는 UI 개선 작업 수행.
